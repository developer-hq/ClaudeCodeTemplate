# Git Flow 分支管理

标准化的分支管理工作流：

## 分支类型
- `feature/` - 新功能开发
- `hotfix/` - 紧急修复
- `release/` - 发布准备
- `bugfix/` - Bug 修复

## 操作选项
1. **创建分支** - 从 main 创建新的功能分支
2. **合并分支** - 将功能分支合并回 main
3. **删除分支** - 清理已完成的分支
4. **同步分支** - 与远程保持同步

## 最佳实践
- 分支命名规范
- 提交信息格式化
- 自动清理合并后的分支
- 保持历史记录清晰

让分支管理变得简单高效！
